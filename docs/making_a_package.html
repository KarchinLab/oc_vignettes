6.1 Learning Objectives

By the end of this tutorial, you should be able to:

  • Create a package on a local installation of OpenCRAVAT
  • Install available annotators from the open cravat store
  • Develop filters for a package
  • Determine layout
  • Upload and annotate variant files with a package from OpenCRAVAT
  • Export annotation results
  • Share package with the OpenCRAVAT community

6.1.1 When to use a package vs. individual annotators

When using OpenCRAVAT, users have the option to leverage pre-packaged annotators or select individual annotators. A package is a module which defines module installation and job parameters. Packages are always the best choice when you want to automate an analysis workflow that you will repeat many times.

Choosing the right approach can significantly impact the efficiency of the analysis. Below we explore the pros of using a package versus individual annotators.

6.1.2 Pros of using a package

  1. Comprehensive Analysis: Packages include multiple annotators, providing a more comprehensive analysis of variants.

  2. Ease of Use: Packages are pre-configured, making them easier to setup without needing to manually select individual annotators.

  3. Consistency: Using a package ensures consistent application of annotations across different analyses.

6.2 How to make a package

For example, if you use the below process often:

  • Install SIFT, BayesDel,CADD Exome, VEST4, PrimateAI,GERP++,FATHMM, and REVEL modules.
  • Annotate with the modules.
  • Filter for Pathogenic variants only.
  • Update layout to user preferences.
  • Make csv and excel reports.
  • Share your package with the OpenCRAVAT community
